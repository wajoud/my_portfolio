## 💻 Quick Start — Local Development

You can run the portfolio in **Unified Mode** (single server) or **Developer Mode** (split frontend/backend servers for code editing).

### Method A: Unified Mode (Recommended)
This runs the entire full-stack application (React frontend + FastAPI backend) under a single command on **Port 8000**:

```bash
# 1. Clone & enter repository
git clone https://github.com/wajoud/my_portfolio.git
cd my_portfolio

# 2. Set up virtual environment & install requirements
python3 -m venv .venv
source .venv/bin/activate          # macOS / Linux
pip install -r requirements.txt

# 3. Start backend server
uvicorn main:app --host 0.0.0.0 --port 8000 --reload
```
> 🌐 Open **[http://localhost:8000](http://localhost:8000)** in your browser. FastAPI automatically serves the pre-compiled, optimized React frontend directly at the root, complete with full contact form operations!

---

### Method B: Developer Mode (Split Servers for UI Editing)
Use this if you are actively editing React code. It enables Hot Module Replacement (HMR) for instant browser refreshes:

1. **Start the backend server on Port 8000** (handles contact form submissions):
   ```bash
   source .venv/bin/activate
   uvicorn main:app --host 0.0.0.0 --port 8000 --reload
   ```

2. **Start the Vite dev server on Port 5173** (in a separate terminal window):
   ```bash
   cd portfolio
   npm install      # First time only
   npm run dev
   ```
   > 🌐 Open **[http://localhost:5173](http://localhost:5173)** in your browser. Vite automatically proxies any backend operations like `/contact` to the FastAPI backend running on port 8000.

## ☁️ Production Deployment

Choose the deployment strategy that best fits your infrastructure needs:

<details>
<summary>📦 <strong>Option A: Virtual Private Server (AWS EC2 / Ubuntu) — Full Control Guide</strong></summary>

<br/>

### Step 1 · Server Setup
```bash
sudo apt update && sudo apt upgrade -y
sudo apt install nginx python3-pip python3-venv certbot python3-certbot-nginx -y
```

### Step 2 · Project Setup & Compilation
Before starting the backend, we must compile the modern React/TypeScript frontend so the assets are built:
```bash
cd ~
git clone https://github.com/wajoud/my_portfolio.git
cd my_portfolio

# 1. Install Node.js (required to compile the React frontend)
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

# 2. Build the optimized React frontend assets
cd portfolio
npm install
npm run build
cd ..

# 3. Create virtual environment & install requirements
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip && pip install -r requirements.txt gunicorn uvicorn
```

### Step 3 · Systemd Service (`/etc/systemd/system/portfolio.service`)
```ini
[Unit]
Description=Portfolio FastAPI App
After=network.target

[Service]
User=ubuntu
Group=www-data
WorkingDirectory=/home/ubuntu/my_portfolio
ExecStart=/home/ubuntu/my_portfolio/.venv/bin/gunicorn \
          -w 2 -k uvicorn.workers.UvicornWorker \
          -b 127.0.0.1:8000 main:app
Restart=always

[Install]
WantedBy=multi-user.target
```
```bash
sudo systemctl daemon-reload
sudo systemctl enable --now portfolio.service
```

### Step 4 · Nginx Config (`/etc/nginx/sites-available/wajoudnoorani.com`)
```nginx
server {
    listen 80;
    server_name wajoudnoorani.com www.wajoudnoorani.com;
    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```
```bash
sudo ln -s /etc/nginx/sites-available/wajoudnoorani.com /etc/nginx/sites-enabled/
sudo nginx -t && sudo systemctl reload nginx
```

### Step 5 · SSL Setup
```bash
sudo ufw allow 'Nginx Full' && sudo ufw --force enable
sudo certbot --nginx -d wajoudnoorani.com -d www.wajoudnoorani.com
```

### 🔄 Update Flow
Run these steps whenever you push new changes to GitHub:
```bash
cd ~/my_portfolio && git pull

# Recompile frontend assets
cd portfolio && npm install && npm run build && cd ..

# Reinstall python dependencies & restart
source .venv/bin/activate
pip install -r requirements.txt
sudo systemctl restart portfolio.service
```

### 📡 Service Management
```bash
sudo systemctl status portfolio.service    # Check status
sudo systemctl restart portfolio.service   # Restart
sudo journalctl -u portfolio.service -f    # Live logs
sudo tail -f /var/log/nginx/error.log      # Nginx errors
```
